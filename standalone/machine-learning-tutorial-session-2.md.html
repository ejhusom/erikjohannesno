Pipeline details

Requirements

  • One of the following operating systems:
    • Linux (RECOMMENDED)
    • macOS
    • Windows with WSL (Windows Subsystem for Linux)
    • Windows (NOT recommended, might be difficult to get things running depending on how you install Python)
  • The following software needs to be installed:
    • Python
    • Git
    • DVC

Overview of approach

  1. Installation
  2. Restructure raw data:
python3 src/fitbit.py [FILEPATH_TO_RAW_DATA]
  1. Place the data files (output of previous step) in assets/data/raw.
  2. Configure parameters in params.yaml.
  3. Run the pipeline:
dvc repro

Installation

  1. Download the source code using git: github.com/ejhusom/bluehealthpass-ml
git clone https://github.com/ejhusom/bluehealthpass-ml.git
  1. Installing required Python-packages:
cd bluehealthpass-ml
mkdir venv
python3 -m venv venv
source venv/bin/activate
pip3 install -r requirements.txt
  1. Set up DVC:
dvc init --no-scm

Data restructuring

  • Run the following command:
python3 src/fitbit.py [FILEPATH_TO_RAW_DATA]

Example:

python3 src/fitbit.py ~/Onedrive/Datasets/Fitbit/BlueHealthPass_CompleteRawDataset
  • Create folder for data: assets/data/raw/[DATASET_NAME]
  • Place the restructured data files in assets/data/raw:
mv *.csv assets/data/raw/[DATASET_NAME]

Balance the distribution

  • Rename files to assign them to the training or test set.
    • A suggestion for the splitting is automatically created by fitbit.py, but it is not robust.
    • This is another suggestion:
      • Training set IDs: 01, 04, 07, 10, 18, 20, 22, 23, 24, 26, 29, 34, 35, 36, 37, 39, 40, 42
      • Test set IDs: 02, 03, 05, 08, 12, 13, 14, 16, 19, 21, 27, 28, 31, 32, 33, 38, 41
    • For the tutorial: Use the files in the zip-folder bhp_restructured.zip

Set up DVC with source code management (SCM):

dvc init

Tracking results with git

  • Run experiments with dvc exp run
  • Plots and DVC log will be tracked by Git
  • Large files (models, profiling reports) will not be tracked by Git
    • DVC handles these files in cache
    • DVC can be synced with cloud storage to keep track of these files
  • See metrics with: dvc metrics show
  • Compare metrics with: dvc metrics diff
  • Compare experiments with: dvc exp show --pcp --all-branches --sort-by r2
